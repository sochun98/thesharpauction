## 2026-04-18 (2)

**Q. 시군구 단위 분할 수집, 상한 증가, GUI(수집+지도), MySQL 저장을 구현하라.**

A. 세 파일로 구현.

**src/db.py** (신규)
- MySQL `auction_cases` 테이블 (35개 컬럼 + lat/lng + 인덱스)
- `upsert_records`: ON DUPLICATE KEY UPDATE (case_no + item_no 기준)
- `load_map_data`, `load_case_detail`, `update_geocode`
- `load_ungeocode_records`: 좌표 없는 레코드 배치 조회

**src/collector.py** (수정)
- `collect_sido_year`: sgg_code 파라미터 추가, 상한 1000페이지, save_fn 콜백
- `collect_by_sigungu`: 시도 내 시군구 목록 × 연도 분할 수집
- `get_sigungu_list(sido_name)`: 시도별 시군구 목록 반환
- CLI: `--sigungu`, `--mysql` 옵션 추가

**src/collector_app.py** (신규) — `streamlit run src/collector_app.py`
- [수집] 탭: 시도/시군구/연도 선택 → collect_by_sigungu → MySQL upsert → 실시간 로그
- [지도] 탭: DB 데이터 조회 → folium 지도 (상태별 색상: 진행중=빨강, 낙찰=파랑, 재매각=주황)
             → 좌표 보완 버튼 (네이버 API 배치 지오코딩)
             → 사건번호 선택 → 상세보기 (Playwright 스크린샷)

**MySQL 환경변수** (`env_thesharpcharm.env`에 추가):
```
MYSQL_HOST=localhost
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=(비밀번호)
MYSQL_DATABASE=auction_db
```

## 2026-04-18 (1)

**Q. 경기도 2026년 수집 결과는 어떻게 되었는가?**

A. 백그라운드 수집 완료. 결과:

| 항목 | 수치 |
|------|------|
| API 응답 총건수 | 34,132건 |
| 중복 제거 후 고유 레코드 | **6,968건** |
| 처리 페이지 | 500페이지 (상한 도달) |
| 소요 시간 | 약 10분 |

중복이 많은 이유: 한 사건에 여러 물건번호(동, 호수)가 있으면 API에서 각각 별도 행으로 반환됨. 34,132 → 6,968로 약 80% 중복.

**문제점**: 500페이지 상한 도달로 수집이 완전하지 않을 수 있음.
→ 시군구 단위로 분할 수집하는 방식으로 개선 필요.

## 2026-04-17 (2)

**Q. 전국, 연도별 기일 지정된 사건을 법원·연도 기준으로 수집하자 (2024·2025·2026년). 수집 항목 구성이 궁금하다.**

A. `src/collector.py`로 구현. 수집 가능한 항목은 다음과 같다.

| 분류 | 항목 |
|------|------|
| 사건 | 사건번호 |
| 법원 | 법원명, 경매계, 법원전화 |
| 주소 | 시도, 시군구, 동, 지번, 건물명, 동호수, 전체주소(표시용) |
| 물건 | 용도(텍스트), 용도대분류코드, 용도중분류코드, 구조·면적, 면적최소·최대 |
| 가격 | 감정가, 최저매각가, 낙찰가, 1~4회차 공고 최저가, 1회차 최저가율 |
| 경매일정 | 매각기일, 매각결정기일, 매각장소, 매각기일횟수 |
| 유찰 | 유찰횟수 |
| 상태 | 물건상태코드(01=진행/04=낙찰/06=재매각 등), 진행여부(Y/N), 물건번호, 목적물번호, 병합사건번호 |

사용법:
```
python src/collector.py --years 2024 2025 2026 --output data/nationwide
python src/collector.py --years 2026 --sido 경기 서울
```

실측 확인: 경기도 2026년 기준 API 총 34,132건 (중복 제거 후 수천 건).

좌표(`wgs84Xcordi/Ycordi`)는 API에서 소수점이 잘려 정수로만 옴 → 정확한 위치는 네이버 지오코딩으로 별도 보완 필요.

## 2026-04-17 (1)

**Q. 오늘까지 만들어진 경매 사건번호를 수집할 수 있는가?**

A. 가능 여부는 단계에 따라 다릅니다.

**✅ 가능: 경매 기일이 지정된 사건번호 전수 수집**

API로 지역 + 기간 조합으로 수집할 수 있습니다.
- `statNum="05"` (전체) + `bidBgngYmd` / `bidEndYmd` 필드로 연도 필터 가능
- 수원 영통구 기준 2026년 경매 기일 있는 사건 → 약 240건 확인됨

**❌ 불가능: 경매 기일 없는 순수 접수 단계 사건번호**

법원이 접수하면 사건번호를 부여하지만, 경매 기일이 지정되기 전까지는 공개 API에 노출되지 않습니다.

실제 흐름:
1. 접수 → 사건번호 부여 (2026타경XXXX)
2. 법원 조사·감정 (수개월)
3. 경매 기일 지정 ← **이 시점부터 API에 노출**
4. 공개 입찰

**현실적으로 구축 가능한 것**

특정 법원 + 연도 기준으로 기일 지정된 사건을 전수 수집하는 기능.
예상 소요 시간:
- 수원지방법원 + 2026년: ~1분
- 경기도 전체 + 2026년: ~10분
- 전국 + 2026년: ~2시간
